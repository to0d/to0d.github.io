Clean up residual tmp_pack files left over by a dead git–pack–objects process

This is clean up of 11 GB of residual tmp_pack_* files that were generated during a 4.5 hour window yesterday and were collectively pushing the repo's perceived size above the size limit.  These tmp_pack files are no longer referenced by any running git pack–objects process, so they are safe to remove with git prune .

For future reference, here is the procedure that @cynthia and I came up with (also listed in https://gitlab.com/gitlab–com/gl–infra/infrastructure/–/issues/13228#note_563112994 ):

&#x000A;
&#x000A; 
# Define the target directory for the bare git repo.

$ GIT_DIR=/var/opt/gitlab/git-data/repositories/@hashed/d8/42/d842432ca69640a34326774932029f8ea41405dac8d56d3ef9ce5385bcaeac32.git

# Confirm that no running processes (e.g. git pack-objects) have an open file descriptor for these temp pack files.

$ sudo lsof -n | grep $( basename $GIT_DIR ) | grep 'tmp_pack' | wc -l

# Show the total size on disk.

$ sudo du -sh $GIT_DIR

# Show the count and size of objects, including a list of garbage packfiles.

$ sudo -i -u git /opt/gitlab/embedded/bin/git --git-dir $GIT_DIR count-objects --verbose --human-readable

# Dry run of pruning the garbage temp pack files.

$ sudo -i -u git /opt/gitlab/embedded/bin/git --git-dir $GIT_DIR prune --verbose --expire='1 hour ago' --dry-run

# Actual run of pruning the garbage temp pack files.

$ sudo -i -u git /opt/gitlab/embedded/bin/git --git-dir $GIT_DIR prune --verbose --expire='1 hour ago'

# Confirm the pack files are removed and the repo size and object counts are still reasonably similar.

$ sudo -i -u git /opt/gitlab/embedded/bin/git --git-dir $GIT_DIR count-objects --verbose --human-readable

# Confirm the disk space usage has been reduced.

$ sudo du -sh $GIT_DIR
